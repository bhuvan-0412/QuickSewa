# Security Policy

## Supported Versions

The following versions of the project are currently being supported with security updates:

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | :white_check_mark: |
| < 0.1   | :x:                |

## Reporting a Vulnerability

We take the security of this project seriously. If you find a security vulnerability, please do NOT report it publicly (such as via a Git issue or public commit message).

Instead, please report security issues responsibly by contacting the maintainers directly or via the designated security team email.

### Process
1. Send an email describing the vulnerability, how it can be reproduced, and any potential impact.
2. We will acknowledge receipt of your vulnerability report within 48 hours.
3. We will work to investigate and fix the vulnerability.
4. We will coordinate a release and publish a security advisory if necessary.
